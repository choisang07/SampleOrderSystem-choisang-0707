# 콘솔 화면 예시 (Screens)

관련 문서: [requirement.md](requirement.md) (기능 명세) · [design.md](design.md) (아키텍처/패키지 구조)

이 문서는 각 메뉴의 콘솔 출력 예시를 정리한 것으로, `View/ConsoleView` 및 `Controller/MainController` 구현 시 화면 구성의 기준으로 삼는다. **예시의 데이터 값(ID 포맷, 수량 등)은 어디까지나 샘플이며, 실제 구현 시 포맷이 달라질 수 있다** — 하단 "ID 포맷 (현재 기준, 확정 아님)" 참고.

## 메인 메뉴

```
__________________________________________________
반도체 시료 생산주문 관리 시스템
__________________________________________________
시스템 현황 2026-07-15 14:22:22

등록 시료    12 종    총 재고    2800ea
전체 주문    25 건    생산라인    8건 대기

[1] 시료 관리    [2] 시료 주문
[3] 주문 승인/거절    [4] 모니터링
[5] 생산라인 조회    [6] 출고 처리
[0]    종료

선택 >
```

- 상단에 현재 시각, 등록 시료 종수, 총 재고 합계, 전체 주문 건수, 생산라인 대기 건수를 요약 표시한다 (requirement.md 5.1절 "전체 시료에 대한 요약 정보").
- 메뉴는 6개 항목(시료 관리 / 시료 주문 / 주문 승인·거절 / 모니터링 / 생산라인 조회 / 출고 처리)으로, requirement.md 5.1절의 "주문(접수/승인/거절)"을 **"시료 주문"과 "주문 승인/거절" 2개 메뉴로 분리**했다 — Phase 2(주문 접수)와 Phase 3(승인/거절)의 경계와 정확히 일치한다.

## [1] 시료 관리

```
__________________________________________________
[1] 시료 관리
__________________________________________________
[1] 시료 등록    [2] 시료 목록    [3] 시료 검색    [0] 뒤로
선택 > 2

등록 시료 목록 (총 12 종)
ID    시료명    평균 생산시간    수율    현재 재고
..
..
..
..
..
외 7종 [P]이전페이지 [N] 다음페이지
```

- 시료 목록은 페이지네이션(`[P]`/`[N]`)을 지원한다 — 한 화면에 5건 표시 후 "외 N종"으로 나머지를 안내.

## [2] 시료 주문

```
__________________________________________________
[2] 시료 주문
__________________________________________________
시료 ID > S-003
고객명 > 삼성전자
주문 수량 > 200

입력 내용 확인

시료 파워 기판-6인치    (S-003)
고객    삼성전자
수량    200 ea

[Y] 예약 접수    [N] 취소
선택 > Y

예약 접수 완료.

주문번호    ORD-20260715-002232
현재 상태 RESERVED
```

- 입력(시료 ID/고객명/수량) 후 확인 화면을 보여주고 `[Y]/[N]`으로 최종 확정한다.
- 주문번호는 `ORD-YYYYMMDD-HHMMSS` 형식(날짜/시간 기반)이다.

## [3] 주문 승인/거절

```
__________________________________________________
[3] 주문 승인/거절
__________________________________________________
승인 대기 중인 예약 목록 (RESERVED)

번호    주문번호    고객    시료    수량    상태
[1] ...
[2] ...
[3] ...

승인할 번호 > 3

재고 확인 중 ..

시료 파워 기판-6인치    현재 재고 30 ea
주문 수량    200 ea    부족분    170 ea <-이 수량만 생산

재고 부족. 부족분 170 ea 승인하시겠습니까? (실생산량 200 ea / 165 min)
[Y] 승인 [N] 주문 거절
선택 > Y

승인 완료.

상태 변경    RESERVED -> PRODUCING
주문번호    ORD-20260715-002232
```

- RESERVED 목록에서 번호로 선택 → 재고 확인 → 부족분/실생산량/총소요시간을 안내하고 `[Y]/[N]`으로 승인/거절.
- "이 수량만 생산"이라는 주석은 재고로 충당 가능한 만큼은 승인 즉시 차감되고 **부족분만 생산 대상**이라는 규칙(requirement.md 5.6절, docs/design.md §6)과 일치한다.

## [5] 생산라인 조회

```
__________________________________________________
[5] 생산라인 조회
__________________________________________________
생산라인 1개 (단일 라인)    현재 상태 : RUNNING

현재 처리 중

주문번호    ORD-20260715-002232     시료     파워 기판-6인치
주문량    80 ea    재고    30 ea -> 부족 50 ea -> 실생산량 54 ea (수율 0.92/49min)
진행        72%    완료 예정(09:49)

대기 중인 주문

순서        주문번호    시료    주문량    부족분    실생산량
1 ..
2 ..
3 ..

* 부족분 = 주문량 - 재고. 실 생산량 = ceil(부족분 / 수율)
선택 >
```

- **"생산라인 1개(단일 라인)"** — docs/design.md §10에서 확정한 "단일 글로벌 FIFO 큐" 결정과 정확히 일치한다.
- 진행률(%)과 완료 예정 시각을 wall-clock 기준으로 표시 (docs/design.md §6 lazy 체크 방식과 일치).
- 하단에 계산 공식(부족분 = 주문량 - 재고, 실생산량 = ceil(부족분/수율))을 화면에 직접 안내.


- CONFIRMED 주문 목록에서 번호 선택 → 출고 처리 완료 안내(주문번호/출고수량/처리일시/상태).
- **주의**: 예시 화면 마지막 "상태 CONFIRMED"는 requirement.md 5.7절("출고 실행 → 주문 상태 RELEASE로 전환") 기준으로는 `RELEASE`가 맞을 것으로 보인다(예시 오타로 추정) — 실제 구현(`ReleaseService::release` → `ConfirmedState::release` → `ReleasedState`)은 문서/design.md 규칙대로 `RELEASE`로 전환하는 것을 기준으로 한다.

## ID 포맷 (현재 기준, 확정 아님)

화면 예시는 샘플 데이터라 정확한 포맷은 구현 중 바뀔 수 있지만, 사용자 확인 결과 현재까지의 방향은 다음과 같다.

| 대상 | 현재 기준 포맷 | 비고 |
|---|---|---|
| 시료 ID | `S-{3자리 순번}` (예: `S-003`) | 기존 docs/design.md §5 예시(`S1`)에서 변경 — §5도 이 포맷으로 갱신 |
| 주문번호 | `ORD-{YYYYMMDD}-{HHMMSS}` (예: `ORD-20260715-002232`) | 날짜/시간 기반. 기존 §5 예시(`O1`)에서 변경 |

- Phase 0에서 만든 Repository 계층(`Json*Repository`)은 ID를 문자열로 저장/조회만 하므로 포맷 변경에 영향받지 않는다.
- 실제 ID 채번 로직은 `SampleFactory`(Phase 1)/`OrderFactory`(Phase 2)가 담당하므로, 그 시점에 이 포맷이 최종 확정되지 않았다면 반드시 재확인한다.

## 참고: 생산라인 조회 화면의 계산 예시

"재고 30 ea -> 부족 50 ea -> 실생산량 54 ea (수율 0.92/49min)"에서 `ceil(50/0.92)=55`가 정확한 계산값이다(54는 예시 화면의 반올림/오타로 추정). requirement.md 5.6절/design.md §6은 `ceil`로 명시되어 있으므로, 실제 구현·테스트케이스 작성 시에는 정확한 `ceil` 계산값을 기준으로 한다(화면 예시의 54라는 숫자를 그대로 테스트 기대값으로 베끼지 않도록 주의).
