# 콘솔 화면 예시 (Screens)

관련 문서: [requirement.md](requirement.md) (기능 명세) · [design.md](design.md) (아키텍처/패키지 구조)

이 문서는 각 메뉴의 콘솔 출력 예시를 정리한 것으로, `View/ConsoleView` 및 `Controller/MainController` 구현 시 화면 구성의 기준으로 삼는다. 예시의 데이터 값(ID, 수량 등)은 임의의 샘플이며, 실제 포맷/ID 체계는 하단 "확인 필요" 참고.

## 메인 메뉴

```
__________________________________________________
반도체 시료 생산주문 관리 시스템
__________________________________________________
시스템 현황 2026-07-15 14:22:22

등록 시료    12 종    총 재고    2800ea
전체 주문    25 건    생산라인    8건 대기

[1] 시료 관리    [2] 시료 주문
[3] 주문 승인/거절    [4] 모니터링
[5] 생산라인 조회    [6] 출고 처리
[0]    종료

선택 >
```

- 상단에 현재 시각, 등록 시료 종수, 총 재고 합계, 전체 주문 건수, 생산라인 대기 건수를 요약 표시한다 (requirement.md 5.1절 "전체 시료에 대한 요약 정보").
- 메뉴는 6개 항목(시료 관리 / 시료 주문 / 주문 승인·거절 / 모니터링 / 생산라인 조회 / 출고 처리)으로, requirement.md 5.1절의 "주문(접수/승인/거절)"을 **"시료 주문"과 "주문 승인/거절" 2개 메뉴로 분리**했다 — Phase 2(주문 접수)와 Phase 3(승인/거절)의 경계와 정확히 일치한다.

## [1] 시료 관리

```
__________________________________________________
[1] 시료 관리
__________________________________________________
[1] 시료 등록    [2] 시료 목록    [3] 시료 검색    [0] 뒤로
선택 > 2

등록 시료 목록 (총 12 종)
ID    시료명    평균 생산시간    수율    현재 재고
..
..
..
..
..
외 7종 [P]이전페이지 [N] 다음페이지
```

- 시료 목록은 페이지네이션(`[P]`/`[N]`)을 지원한다 — 한 화면에 5건 표시 후 "외 N종"으로 나머지를 안내.

## [2] 시료 주문

```
__________________________________________________
[2] 시료 주문
__________________________________________________
시료 ID > S-003
고객명 > 삼성전자
주문 수량 > 200

입력 내용 확인

시료 파워 기판-6인치    (S-003)
고객    삼성전자
수량    200 ea

[Y] 예약 접수    [N] 취소
선택 > Y

예약 접수 완료.

주문번호    0RD-20260715-002232
현재 상태 RESERVED
```

- 입력(시료 ID/고객명/수량) 후 확인 화면을 보여주고 `[Y]/[N]`으로 최종 확정한다.
- 주문번호는 `ORD-YYYYMMDD-HHMMSS` 형식으로 보인다(예시는 `0RD-...`로 표기되어 있으나 오타로 추정 — 하단 "확인 필요" 참고).

## [3] 주문 승인/거절

```
__________________________________________________
[3] 주문 승인/거절
__________________________________________________
승인 대기 중인 예약 목록 (RESERVED)

번호    주문번호    고객    시료    수량    상태
[1] ...
[2] ...
[3] ...

승인할 번호 > 3

재고 확인 중 ..

시료 파워 기판-6인치    현재 재고 30 ea
주문 수량    200 ea    부족분    170 ea <-이 수량만 생산

재고 부족. 부족분 170 ea 승인하시겠습니까? (실생산량 200 ea / 165 min)
[Y] 승인 [N] 주문 거절
선택 > Y

승인 완료.

상태 변경    RESERVED -> PRODUCING
주문번호    0RD-20260715-002232
```

- RESERVED 목록에서 번호로 선택 → 재고 확인 → 부족분/실생산량/총소요시간을 안내하고 `[Y]/[N]`으로 승인/거절.
- "이 수량만 생산"이라는 주석은 재고로 충당 가능한 만큼은 승인 즉시 차감되고 **부족분만 생산 대상**이라는 규칙(requirement.md 5.6절, docs/design.md §6)과 일치한다.

## [5] 생산라인 조회

```
__________________________________________________
[5] 생산라인 조회
__________________________________________________
생산라인 1개 (단일 라인)    현재 상태 : RUNNING

현재 처리 중

주문번호    0RD-20260715-002232     시료     파워 기판-6인치
주문량    80 ea    재고    30 ea -> 부족 50 ea -> 실생산량 54 ea (수율 0.92/49min)
진행        72%    완료 예정(09:49)

대기 중인 주문

순서        주문번호    시료    주문량    부족분    실생산량
1 ..
2 ..
3 ..

* 부족분 = 주문량 - 재고. 실 생산량 = ceil(부족분 / 수율)
선택 >
```

- **"생산라인 1개(단일 라인)"** — docs/design.md §10에서 확정한 "단일 글로벌 FIFO 큐" 결정과 정확히 일치한다.
- 진행률(%)과 완료 예정 시각을 wall-clock 기준으로 표시 (docs/design.md §6 lazy 체크 방식과 일치).
- 하단에 계산 공식(부족분 = 주문량 - 재고, 실생산량 = ceil(부족분/수율))을 화면에 직접 안내.

(예시 화면 미제공 — requirement.md 5.7절 기준으로 CONFIRMED 주문 목록에서 선택 → RELEASE 전환하는 흐름으로 추정. 실제 화면 예시 확보 시 이 문서에 추가한다.)

## 확인 필요 (구현 전 확정할 것)

1. **주문번호 포맷**: 예시가 `0RD-20260715-002232`(숫자 0으로 시작)인데, `ORD-YYYYMMDD-HHMMSS`(영문 O)의 오타인지 확인 필요. 확정되면 기존 docs/design.md §5의 단순 순번 ID(`O1`, `O2`)에서 **날짜/시간 기반 ID**로 스키마를 변경해야 한다(Phase 0에 이미 만들어둔 Repository 계층은 ID를 그대로 저장만 하므로 이 변경에 영향받지 않지만, ID 생성 로직을 담당할 Phase 2의 `OrderFactory`는 이 포맷을 따라야 한다).
2. **시료 ID 포맷**: 예시가 `S-003`(하이픈 포함)인데, docs/design.md §5 스키마는 `S1`(하이픈 없음)이다. 어느 쪽이 맞는지 확인 필요 — `SampleFactory`(Phase 1)의 ID 채번 규칙에 영향.
3. **생산라인 조회 화면의 계산 예시 오차**: "재고 30 ea -> 부족 50 ea -> 실생산량 54 ea (수율 0.92)"는 `ceil(50/0.92)=55`가 되어야 할 것 같다(54는 반올림/버림 오차로 보임). 실제 반올림 규칙이 `ceil`이 맞는지 재확인(요구사항 5.6절/design.md §6은 `ceil`로 명시되어 있어 이 예시 화면 쪽의 오타로 추정).
4. **출고 처리(`[6]`) 화면 예시**가 없어서 다른 메뉴와 톤을 맞춰 임의로 만들어야 하는 상태.
